# VP-041: Single-Level Monotonicity

## Property Statement

Raising exactly one goal's Implementation Level (per the ordinal order NI < PI < LI < FI), with all other inputs fixed and the goal applicable in both states, never decreases the overall percent or the goal's function sub-score.

## Source Contract

- **BC:** BC-4.04.001 — Weighted Percent Computation
- **Postcondition/Invariant:** Invariant 2 (monotonicity)

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| kani | Kani | yes — ≤64 goals | symbolic pair of states differing in one level |

## Proof Harness Skeleton

```rust
#[kani::proof]
fn verify_vp_041() {
    let (catalog, answers, idx): (BoundedCatalog, BoundedAnswers, usize) = kani::any();
    kani::assume(catalog.is_valid() && idx < catalog.goal_count());
    let raised = answers.with_raised_level(idx);
    kani::assume(raised.is_some()); // not already FI, not N/A
    let p0 = score(&catalog, &answers, &none()).percent.unwrap();
    let p1 = score(&catalog, &raised.unwrap(), &none()).percent.unwrap();
    assert!(p1 >= p0);
}
```
