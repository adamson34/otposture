# S-6.03: Report Output Safety, Offline Harness & Release Packaging

## Narrative
- **As an** OT buyer, **I want** auditable proof the binary never speaks to a network and never half-writes a report, **so that** the trust claims are evidence, not marketing.

## Acceptance Criteria

### AC-001 (traces to BC-5.05.004 postconditions 1–4, VP-076)
- Report writing: pre-flight path check; exists without --force → E-RPT-001; directory → E-RPT-002; inside store → E-RPT-003 (wins over --force); atomic temp+rename (old-complete or new-complete under any fault); default name `otposture-report-<date>-s<seq>.html`, collision-resistant, no PII.
- **Test:** `test_BC_5_05_004_output_safety()` + fault injection (VP-076)

### AC-002 (traces to BC-6.06.003 postconditions 1–2, VP-083)
- CI syscall-trace harness runs the full integration suite asserting zero network syscalls (Linux strace filter; macOS dtrace; Windows documented manual gate per coverage-matrix Gap Register); suite re-run with interfaces down produces identical outputs/exit codes.
- **Test:** CI job `offline-trace` + airgap-parity fixture run

### AC-003 (traces to BC-6.06.003 invariants 1–2, VP-084)
- cargo-deny ban list final (reqwest/hyper/ureq/curl/tokio-net/async-std-net/native-tls families); no URL-constructing code (lint/grep gate); proxy env vars demonstrably ignored.
- **Test:** `test_BC_6_06_003_env_independence()` + deny CI (existing from S-1.01, now against the real dependency tree)

### AC-004 (traces to ADR-008 / NFR-005, ASM-007)
- Release CI builds 4 static targets (musl x86_64/aarch64, macOS, Windows +crt-static); binary < 25 MB; cold start < 200 ms benchmark; SHA-256 checksums published; portable-execution smoke test (no admin, copy-run).
- **Test:** release workflow gates
