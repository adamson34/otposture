# S-1.03: Weighted Percent & Function Sub-Scores

## Narrative
- **As a** plant engineer, **I want** my answers scored by one exact, inspectable formula, **so that** the percentage I report upward can be re-derived by hand.

## Acceptance Criteria

### AC-001 (traces to BC-4.04.001 postconditions 1–2, DI-012)
- `percent = Σ(w·value)/Σ(w)` over applicable (non-N/A) goals at full rational precision; empty applicable set → `percent: None` ("not scoreable"), never 0 or 1.
- **Test:** `test_BC_4_04_001_weighted_percent_formula()`, `test_BC_4_04_001_all_na_not_scoreable()`

### AC-002 (traces to BC-4.04.001 invariants 2–3)
- Bounds [0,1]; percent = 1 iff all applicable FI; raising one level never lowers percent.
- **Test:** proptest `prop_BC_4_04_001_bounds_monotonic()` + Kani harnesses VP-040/VP-041

### AC-003 (traces to BC-4.04.002 postconditions 1–2)
- Per-function sub-scores via the identical formula; all-N/A functions report None and are excluded from the roll-up; decomposition identity holds exactly (overall = weighted recombination of function scores).
- **Test:** `test_BC_4_04_002_function_subscores()` + Kani VP-044

### AC-004 (traces to BC-4.04.001 EC-004 / E-SCORE-001)
- Weight ≤ 0 reaching the engine asserts E-SCORE-001 (internal) — unreachable via validated catalogs.
- **Test:** `test_BC_4_04_001_zero_weight_asserts()`

### AC-005 (traces to BC-4.04.001 canonical vectors)
- All four canonical test vectors from BC-4.04.001 pass as golden tests, including (1 + 1/3)/2 = 2/3 exactly.
- **Test:** `test_BC_4_04_001_canonical_vectors()`
