# S-3.04: Catalog Migration — Back-Cast & Series Breaks

## Narrative
- **As a** user when CPG 2.1 ships, **I want** my trend to survive the framework upgrade honestly, **so that** history is never silently rescored and never silently broken.

## Acceptance Criteria

### AC-001 (traces to BC-2.02.006 postconditions 1–3, VP-019/020)
- `migrate` with mapping: back-cast Scores stored alongside originals keyed by seq (originals byte-identical — VP-019), back-cast uses only mapped answers + new catalog (VP-020), assessment remapped, orphans preserved with notes (DEC-012), migration record (version pair, mapping digest, disposition counts); idempotent re-run.
- **Test:** `test_BC_2_02_006_backcast()` + proptest VP-019/020

### AC-002 (traces to BC-2.02.006 EC-001/002/004)
- Splits back-cast to all children with derived-from-split markers + re-assess flags; orphaned overrides reported, never dropped; `--mapping` + `--series-break` together = usage error (exit 2).
- **Test:** `test_BC_2_02_006_edge_cases()`

### AC-003 (traces to BC-2.02.007 postconditions 1–3, VP-021)
- No mapping: interactive confirm or `--series-break` flag records SeriesBreak (at_seq, versions, reason); scripted without flag → E-VAL-010; breaks render as discontinuities; no comparability path crosses an unbridged break (VP-021); later mapping bridges (break record remains, marked bridged).
- **Test:** `test_BC_2_02_007_breaks()` + proptest VP-021

### AC-004 (traces to BC-2.02.007 invariant 1)
- Break records are append-only history; multiple breaks segment the trend correctly.
- **Test:** `test_BC_2_02_007_multiple_breaks_segments()`
