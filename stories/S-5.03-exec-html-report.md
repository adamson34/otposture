# S-5.03: Executive HTML Report

## Narrative
- **As a** COO with 60 seconds, **I want** one self-contained page with the band, the trend, what changed, and what's next, **so that** the security update reads itself.

## Acceptance Criteria

### AC-001 (traces to BC-5.05.003 postcondition 1 — all 8 sections in order)
- (a) site + date + catalog version; (b) headline band + trend arrow + percent sub-metric, floor-cap banner with gaps when active; (c) what changed — driver-attributed movers incl. regressions; (d) top open risks — critical gaps + lowest functions; (e) next best actions with effort labels (floor-gating first); (f) completeness caveat when unknowns > 0; (g) active overrides with rationales; (h) footer — tool version, snapshot seqs, catalog digest prefix.
- **Test:** golden `test_BC_5_05_003_sections()` (VP-074: numbers equal status/diff for same snapshots)

### AC-002 (traces to BC-5.05.003 postconditions 2–3, VP-073)
- Single .html, inline CSS, zero external resources (CI audit), legible at A4/Letter print width; no JS required for content; no personal data (DI-013).
- **Test:** `test_BC_5_05_003_self_contained()` (HTML parse + resource audit)

### AC-003 (traces to BC-5.05.003 invariant 3, EC-004, VP-075)
- All store-sourced strings HTML-escaped (askama autoescape + fuzz store-content→render→parse finds no markup injection); uncapped band never shown without its cap.
- **Test:** fuzz `fuzz_report_escaping` + `test_no_uncapped_without_cap()`

### AC-004 (traces to BC-5.05.003 precondition 1, EC-001..003)
- Snapshot-based only (warns when Assessment is dirty — PRD OQ5); baseline framing for single-snapshot; unknown-heavy caveat box precedes any percentage; bridged-migration derivation note in (c).
- **Test:** `test_BC_5_05_003_report_states()`
