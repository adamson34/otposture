# S-2.01: Catalog Parser & Eight-Rule Validation

## Narrative
- **As the** scoring engine, **I want** only structurally valid catalogs to exist downstream of load, **so that** every later invariant can assume a well-formed catalog.

## Acceptance Criteria

### AC-001 (traces to BC-1.01.001 postconditions 1–3)
- `parse(&str)` losslessly loads catalog TOML into op-score types: id, semver, provenance, ordered functions, goals with all fields, band list (N ≥ 2, N−1 thresholds); unknown fields ignored with debug note; newer schema → E-CAT-002.
- **Test:** `test_BC_1_01_001_parse_lossless()` + proptest round-trip (VP-001)

### AC-002 (traces to BC-1.01.002 postcondition 3, all 8 rules)
- `validate()` enforces: (a) case-insensitive-unique goal IDs; (b) function refs resolve; (c) 0 < weight ≤ 10⁶; (d) tier ∈ {Critical,High,Standard}; (e) band list N ≥ 2, strictly increasing interior thresholds in (0,1); (f) non-empty provenance; (g) valid semver; (h) impact/effort present.
- **Test:** `test_BC_1_01_002_each_rule()` (one fixture per rule) + Kani VP-004 (no false accept)

### AC-003 (traces to BC-1.01.002 postcondition 2, EC-001)
- All defects reported in one pass, stable order, each naming rule + element + expected condition (E-CAT-003 shape).
- **Test:** `test_BC_1_01_002_multi_defect_itemized()` (VP-003 order-independence proptest)

### AC-004 (traces to BC-1.01.001 VP-002)
- Parser never panics on arbitrary bytes.
- **Test:** fuzz target `fuzz_catalog_parse`
