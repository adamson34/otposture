# S-4.02: Guided Assessment Session

## Narrative
- **As a** time-poor engineer, **I want** a guided walk through unanswered goals that survives an SSH drop, **so that** assessment fits in stolen minutes.

## Acceptance Criteria

### AC-001 (traces to BC-3.03.002 postcondition 1)
- Session presents goals filtered by `--function <FN>` ∩ `--only <unknown|all>` (default unknown), catalog order; displays official title, outcome text, OT note verbatim (never paraphrased — invariant 2); `stale` token does not exist in v1.
- **Test:** `test_BC_3_03_002_filtering_order()` (pure state machine — no TTY needed)

### AC-002 (traces to BC-3.03.002 postconditions 2–3, VP-032)
- Pure `step(state, input) -> (state, effects)` machine: each accepted answer emits a persist effect executed durably before the next prompt; kill at any point loses at most the in-flight prompt (monotone durability — VP-032 kill-injection); end summary shows answered/skipped/remaining + score preview.
- **Test:** `test_BC_3_03_002_state_machine()` + process kill-injection harness

### AC-003 (traces to BC-3.03.002 EC-001/003, invariant 1)
- Nothing-to-assess hints `--only all`; `back` revisits with replace semantics; skip/quit write nothing.
- **Test:** `test_BC_3_03_002_navigation()`

### AC-004 (traces to BC-3.03.002 precondition 1, EC-004)
- Non-TTY invocation → E-VAL-007 pointing to `answer`; 80-col rendering wraps readably (snapshot test).
- **Test:** `test_BC_3_03_002_tty_gate()` + render snapshots
