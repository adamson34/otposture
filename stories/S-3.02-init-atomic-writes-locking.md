# S-3.02: Store Init, Atomic Writes & Locking

## Narrative
- **As a** plant engineer whose workstation loses power, **I want** every store operation to be all-or-nothing, **so that** months of history can't be corrupted by one bad moment.

## Acceptance Criteria

### AC-001 (traces to BC-2.02.002 postconditions + invariants, VP-012)
- All mutations: write-temp → fsync → rename (+ dir fsync where honored); fault injected at every syscall point leaves prior state loadable; no code path truncates in place (VP-013 audit + lint).
- **Test:** fault-injection harness `fault_BC_2_02_002_crash_consistency()` (every op × every fault point)

### AC-002 (traces to BC-2.02.001 postconditions 1–3, VP-010/011)
- `init` creates marker, site (control-char-rejected name → E-VAL-002), empty assessment pinned to (id, version, digest), schema marker — atomically as a unit; refuses existing store (E-STORE-001, byte-identical store after); never touches unrelated files; interrupted init re-runnable.
- **Test:** `test_BC_2_02_001_init()` + fault injection

### AC-003 (traces to BC-2.02.005 postconditions 1–3, VP-018)
- Mutating commands take advisory lock (PID/host/start, O_EXCL create); live-holder contention → E-LOCK-001 fast-fail; dead-PID same-host or age-threshold locks break with notice; read-only commands never lock.
- **Test:** process-level `test_BC_2_02_005_single_writer()` + EC fixtures

### AC-004 (traces to BC-2.02.002 EC-001..003)
- ENOSPC → E-FS-002 original intact; sync-tool rename lock → bounded retry then E-FS-003; orphan temp files cleaned with notice on next load.
- **Test:** `test_BC_2_02_002_edge_cases()`
