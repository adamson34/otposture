# S-4.01: Answer Recording & Unknown Semantics

## Narrative
- **As a** plant engineer, **I want** to record one answer in seconds with typos caught, **so that** updating posture costs nothing.

## Acceptance Criteria

### AC-001 (traces to BC-3.03.001 postconditions 1–3)
- `record_answer`: valid level tokens only (E-VAL-006 with token list incl. unknown-vs-na distinction); replace semantics with refreshed `answered_at` (timestamp passed in, never clocked in pure code); durable before return; explicit-unknown recorded distinct from never-answered (score-identical, flag-distinct — VP-034).
- **Test:** `test_BC_3_03_001_record()` (VP-031 token proptest)

### AC-002 (traces to BC-3.03.001 EC-001/002/004, VP-030)
- Goal-ID normalization ("1a", "1-A" → "1.A" when unambiguous); unknown IDs → E-VAL-004 with nearest-match suggestions; note > 16 KiB → E-VAL-005; assessment never contains orphan goal_id (VP-030 proptest).
- **Test:** `test_BC_3_03_001_normalization_suggestions()`

### AC-003 (traces to BC-3.03.003 postconditions 1–3)
- Unanswered goals score as Unknown with full denominator weight (never N/A-style exclusion); unknown counts surface in all views; Critical+Unknown gates the floor (integration with S-1.04).
- **Test:** `test_BC_3_03_003_unknown_default()` (VP-033 monotone-add proptest)

### AC-004 (traces to BC-3.03.003 EC-001..003)
- 34/34-unknown store scores 0%/Foundational honestly; `skip` is not a level token (error explains unknown vs na).
- **Test:** `test_BC_3_03_003_zero_state()`
