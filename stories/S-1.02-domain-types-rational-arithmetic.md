# S-1.02: Domain Types & Rational Arithmetic

## Narrative
- **As an** implementer of every later story, **I want** the shared domain types and exact-rational number type in op-score, **so that** all subsystems speak one vocabulary and no float ever enters a scoring path.

## Acceptance Criteria

### AC-001 (traces to BC-4.04.005 invariant 1 / DI-012)
- `Rational` wraps `Ratio<i128>`; `Level::{NotImplemented, Partially, Largely, Fully, NotApplicable, Unknown}` with `value()` returning exactly 0, 1/3, 2/3, 1, (none), 0; display formatting renders 1/3 as "0.33"/"33%" only at presentation calls.
- **Test:** `test_BC_4_04_005_level_values_exact_rationals()`

### AC-002 (traces to BC-4.04.005 postcondition 2)
- op-score exposes `ENGINE_VERSION: &str`; types `Catalog, Goal, Function, Band, Tier, AnswerSet, TierOverrides, ScoreResult, Delta, Driver, SnapshotMeta, SeriesBreak` exist with serde derives and no I/O imports.
- **Test:** `test_BC_4_04_005_engine_version_exposed()` + purity lint (S-1.01 AC-002 now non-trivially exercised)

### AC-003 (traces to BC-4.04.005 invariant 1)
- No `f64`/`f32` in any op-score scoring type or function signature; clippy `float_arithmetic` deny passes.
- **Test:** CI clippy gate (red if a float sneaks in)

### AC-004 (traces to BC-4.04.003 precondition 1 — types only)
- `Bands` type enforces construction invariants at the type level: N ≥ 2, N−1 strictly increasing thresholds in (0,1) exclusive; invalid construction returns typed errors (consumed by catalog validation in S-2.01).
- **Test:** `test_bands_construction_rejects_invalid()`
