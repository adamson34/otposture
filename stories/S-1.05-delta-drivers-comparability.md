# S-1.05: Delta, Driver Attribution & Comparability Guard

## Narrative
- **As a** plant engineer, **I want** "▲ +4%" to name exactly which goals moved it, **so that** my upward report survives "says who?".

## Acceptance Criteria

### AC-001 (traces to BC-4.04.006 postconditions 1–2)
- `delta()` produces percent_delta, band_change (incl. floor states), Drivers (every changed-level goal, signed points = w·Δvalue/Σw_to), and composition_adjustment for N/A transitions; Σ drivers + composition = percent_delta exactly.
- **Test:** `test_BC_4_04_006_delta_reconciles()` + Kani VP-053 (the flagship proof)

### AC-002 (traces to BC-4.04.006 postcondition 3, DEC-008)
- `display_round()` allocates whole-percent rounding residue largest-remainder so displayed drivers sum to displayed total.
- **Test:** proptest `prop_BC_4_04_006_display_reconciles()` (VP-054)

### AC-003 (traces to BC-4.04.006 invariants 1–2, EC-002/EC-004)
- Drivers sorted by |points| desc, regressions equal prominence; empty drivers iff no change; floor-only band change reported with override/critical citation.
- **Test:** `test_BC_4_04_006_driver_ordering_regressions()` (VP-055)

### AC-004 (traces to BC-4.04.007 postconditions 1–2, invariant 1)
- `check_comparability()` is the only path to delta(): admits same (id,version,digest) or back-cast-bridged pairs with no unbridged break between; refusals carry typed reason (version/digest/break) for E-SCORE-002 rendering.
- **Test:** `test_BC_4_04_007_guard()` + Kani VP-056; module privacy makes delta() unreachable except via guard (VP-057 lint/review)

### AC-005 (traces to BC-4.04.006/007 canonical vectors)
- Canonical vectors pass, incl. interface-definitions.md's machine-verified example (1/33 + 1/99 = 4/99) as a golden.
- **Test:** `test_BC_4_04_006_canonical_vectors()`
